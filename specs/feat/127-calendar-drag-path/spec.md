# Feature Specification: 캘린더 경로 기반 드래그 선택

**Feature Branch**: `127-calendar-drag-path`
**Created**: 2026-03-29
**Status**: Draft
**Input**: 캘린더 선택 로직 변경 - 사각형 영역 선택에서 마우스가 지나간 경로만 선택되는 방식으로 변경

## User Scenarios & Testing _(mandatory)_

### User Story 1 - 마우스 드래그로 개별 날짜 경로 선택 (Priority: P1)

호스트 또는 참여자가 캘린더에서 날짜를 선택할 때, 마우스를 클릭한 채로 드래그하면 마우스가 실제로 지나간 날짜 셀만 선택된다. 기존처럼 시작점과 끝점 사이의 전체 범위가 선택되는 것이 아니라, 마우스 포인터가 통과한 셀만 개별적으로 선택 상태가 토글된다.

**Why this priority**: 이 기능의 핵심 변경사항이며, 사용자가 원하는 날짜만 정확하게 선택할 수 있도록 하는 기본 인터랙션이다.

**Independent Test**: 캘린더에서 마우스를 드래그하여 비연속적인 경로를 따라 이동했을 때, 경로 위의 셀만 선택되는지 확인

**Acceptance Scenarios**:

1. **Given** 아무 날짜도 선택되지 않은 캘린더, **When** 사용자가 3월 5일을 클릭한 채 3월 6일, 3월 7일을 거쳐 드래그, **Then** 3월 5일, 6일, 7일만 선택됨
2. **Given** 아무 날짜도 선택되지 않은 캘린더, **When** 사용자가 3월 5일(월)을 클릭한 채 3월 12일(월)까지 아래로 드래그(중간에 6일~11일을 지나지 않음), **Then** 마우스가 실제로 지나간 셀만 선택됨 (기존처럼 5일~12일 전체가 선택되지 않음)
3. **Given** 3월 5일, 6일, 7일이 이미 선택된 캘린더, **When** 사용자가 3월 5일을 클릭한 채 3월 6일, 7일을 거쳐 드래그, **Then** 3월 5일, 6일, 7일이 선택 해제됨

---

### User Story 2 - 터치 드래그로 개별 날짜 경로 선택 (Priority: P1)

모바일 사용자가 터치 드래그로 날짜를 선택할 때도 동일하게 손가락이 지나간 경로의 셀만 선택/해제된다.

**Why this priority**: 서비스의 주요 사용자층이 모바일이므로 터치 인터랙션도 동일한 우선순위로 지원되어야 한다.

**Independent Test**: 모바일 기기에서 터치 드래그로 날짜를 선택했을 때 경로 위의 셀만 선택되는지 확인

**Acceptance Scenarios**:

1. **Given** 아무 날짜도 선택되지 않은 캘린더, **When** 사용자가 터치로 3월 5일부터 3월 7일까지 드래그, **Then** 터치가 지나간 셀만 선택됨
2. **Given** 일부 날짜가 선택된 캘린더, **When** 사용자가 선택된 날짜를 터치 드래그, **Then** 터치가 지나간 선택된 셀이 해제됨

---

### User Story 3 - 드래그 중 시각적 피드백 (Priority: P2)

사용자가 드래그하는 동안, 마우스/터치가 지나간 셀에 즉시 시각적 피드백이 표시되어 어떤 날짜가 선택/해제될 것인지 미리 알 수 있다.

**Why this priority**: 사용자 경험 측면에서 중요하지만 핵심 선택 로직보다는 부가적이다.

**Independent Test**: 드래그 중 마우스가 지나간 셀에 하이라이트가 표시되는지 확인

**Acceptance Scenarios**:

1. **Given** 드래그가 시작된 상태, **When** 마우스가 새로운 셀 위를 지나감, **Then** 해당 셀에 즉시 시각적 하이라이트가 표시됨
2. **Given** 드래그 중 시각적 피드백이 표시된 상태, **When** 마우스 버튼을 놓음, **Then** 피드백이 사라지고 최종 선택 상태가 반영됨

---

### Edge Cases

- 드래그 중 마우스가 캘린더 영역 밖으로 나가면? → 드래그가 즉시 종료되고 경로의 선택/해제가 확정됨
- 드래그 중 비활성화된(과거) 날짜 위를 지나가면? → 비활성화된 셀은 무시하고 선택/해제하지 않음
- 매우 빠르게 드래그하여 중간 셀을 건너뛰면? → 마우스 이벤트가 발생한 셀만 선택됨 (이벤트가 발생하지 않은 셀은 선택되지 않음)
- 단일 셀 클릭(드래그 없이)? → 기존과 동일하게 해당 셀의 선택 상태가 토글됨
- 드래그 시작 셀이 선택된 상태이면? → 드래그 경로 위의 셀들이 모두 해제됨 (시작 셀의 상태 기준으로 토글 방향 결정)
- 드래그 시작 셀이 미선택 상태이면? → 드래그 경로 위의 셀들이 모두 선택됨

## Requirements _(mandatory)_

### Functional Requirements

- **FR-001**: 시스템은 마우스 드래그 시 마우스 포인터가 실제로 지나간 날짜 셀만 선택/해제해야 한다
- **FR-002**: 시스템은 터치 드래그 시 터치 포인트가 실제로 지나간 날짜 셀만 선택/해제해야 한다
- **FR-003**: 드래그 시작 시 첫 번째 셀의 현재 선택 상태에 따라 토글 방향이 결정되어야 한다 (선택됨 → 해제 모드, 미선택 → 선택 모드)
- **FR-004**: 드래그 중 각 셀을 지날 때 즉시 시각적 피드백이 제공되어야 한다
- **FR-005**: 비활성화된 날짜(과거 날짜)는 드래그 경로에 포함되더라도 선택/해제되지 않아야 한다
- **FR-006**: 단일 클릭(드래그 없음)은 기존과 동일하게 해당 셀의 선택 상태를 토글해야 한다
- **FR-007**: 드래그 경로에서 이미 한 번 지나간 셀을 다시 지나가더라도, 해당 셀의 상태는 한 번만 변경되어야 한다 (같은 드래그 내에서 이중 토글 방지)

## Success Criteria _(mandatory)_

### Measurable Outcomes

- **SC-001**: 사용자가 비연속적인 날짜를 선택할 때 드래그 한 번으로 원하는 날짜만 정확하게 선택할 수 있다
- **SC-002**: 드래그 중 시각적 피드백이 셀 진입 즉시(100ms 이내) 표시된다
- **SC-003**: 마우스 및 터치 인터랙션 모두에서 동일한 경로 기반 선택 동작이 지원된다
- **SC-004**: 기존 단일 클릭 토글 기능이 변경 없이 유지된다

## Assumptions

- 드래그 중 같은 셀을 여러 번 지나가는 경우, 처음 진입 시에만 토글이 적용된다 (재진입 시 추가 토글 없음)
- 토글 방향은 드래그 시작 셀의 상태를 기준으로 결정되며, 드래그 중간에 변경되지 않는다
- 호스트 날짜 선택, 참여자 투표, 참여자 수정 등 캘린더를 사용하는 모든 화면에 동일하게 적용된다

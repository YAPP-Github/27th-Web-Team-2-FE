# Feature Specification: 날짜 연속 클릭 시 선택 상태 롤백/미입력 버그 수정

**Feature Branch**: `118-date-click-race-fix`
**Created**: 2026-03-23
**Status**: Draft
**Input**: User description: "날짜를 연속해서 클릭하는 경우 미입력되거나 이전에 선택한 날짜가 롤백되는 문제가 있음."

## User Scenarios & Testing _(mandatory)_

### User Story 1 - 사용자가 빠르게 여러 날짜를 연속 선택 (Priority: P1)

참여자가 투표 날짜를 선택하거나, 호스트가 모임 생성 시 후보 날짜를 설정할 때, 빠르게 연속으로 탭/클릭하여 선택하면 모든 선택이 정확히 반영되어야 한다. 현재는 빠르게 클릭하면 일부 날짜가 선택되지 않거나, 이전에 선택했던 날짜가 해제되는 현상이 모바일(터치)과 데스크톱(마우스) 환경 모두에서 발생한다.

**Why this priority**: 핵심 기능인 날짜 선택의 신뢰성에 직접적으로 영향을 미치는 버그. 호스트/참여자 모두에게 영향을 주며, 잘못된 날짜 설정 및 투표 결과로 이어진다.

**Independent Test**: 캘린더에서 5개 이상의 날짜를 빠르게(0.5초 이내 간격으로) 연속 클릭한 후, 모든 날짜가 정상적으로 선택 상태를 유지하는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 참여자가 투표 페이지에 접근한 상태, **When** 3개의 날짜를 0.3초 간격으로 빠르게 연속 클릭, **Then** 3개 날짜 모두 선택 상태로 정확히 표시됨
2. **Given** 이미 2개의 날짜가 선택된 상태, **When** 추가로 3개의 날짜를 빠르게 연속 클릭, **Then** 기존 2개 + 추가 3개 = 총 5개 날짜가 모두 선택 상태로 유지됨
3. **Given** 5개의 날짜가 선택된 상태, **When** 그 중 2개를 빠르게 연속 클릭하여 해제, **Then** 정확히 해당 2개만 해제되고 나머지 3개는 선택 상태 유지

---

### User Story 2 - 참여자가 드래그로 날짜 범위 선택 (Priority: P2)

참여자가 캘린더에서 터치 드래그 또는 마우스 드래그를 통해 날짜 범위를 선택할 때, 선택 범위가 정확히 반영되어야 한다.

**Why this priority**: 드래그 선택은 모바일에서 주요 인터랙션 방식이며, 빠른 드래그 시 동일한 상태 동기화 문제가 발생할 수 있다.

**Independent Test**: 캘린더에서 시작 날짜를 터치/클릭한 후 끝 날짜까지 드래그하여 범위 내 모든 날짜가 정확히 선택되는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 빈 캘린더 상태, **When** 1월 5일에서 1월 10일까지 드래그, **Then** 1월 5일~10일 (6일) 모두 선택됨
2. **Given** 드래그 중인 상태, **When** 빠르게 드래그 방향을 변경, **Then** 최종 드래그 범위만 정확히 반영됨

---

### User Story 3 - 날짜 선택 후 즉시 투표 제출 (Priority: P2)

참여자가 마지막 날짜를 선택한 직후 바로 "투표하기" 버튼을 눌렀을 때, 마지막으로 선택한 날짜가 정확히 포함된 상태로 투표가 제출되어야 한다.

**Why this priority**: 날짜 선택 직후 제출 시 마지막 선택이 누락되면 잘못된 투표 결과가 서버에 저장된다.

**Independent Test**: 날짜를 선택한 직후(1초 이내) 투표하기 버튼을 눌러, 선택한 모든 날짜가 API 요청에 포함되는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 2개의 날짜가 선택된 상태, **When** 3번째 날짜를 선택하고 즉시 투표하기 버튼 클릭, **Then** 3개 날짜 모두 포함된 투표가 서버에 전송됨

---

### Edge Cases

- 같은 날짜를 빠르게 두 번 클릭하면 어떻게 되는가? (선택 → 해제가 정확히 동작해야 함)
- 터치와 마우스 이벤트가 혼합되는 하이브리드 디바이스에서의 동작
- 드래그 시작 후 캘린더 영역 밖으로 포인터가 벗어난 경우
- 네트워크 지연 중 투표 제출 버튼을 여러 번 클릭하는 경우 (기존 isSubmitting 가드로 처리됨)

## Requirements _(mandatory)_

### Functional Requirements

- **FR-001**: 시스템은 연속적인 날짜 클릭 시 모든 클릭에 대해 정확한 선택/해제 토글을 보장해야 한다
- **FR-002**: 시스템은 날짜 선택 상태의 시각적 피드백이 내부 상태와 항상 일치해야 한다
- **FR-003**: 시스템은 드래그 인터랙션 중 시작~끝 범위 내 모든 날짜를 정확히 선택해야 한다
- **FR-004**: 시스템은 날짜 선택 직후 투표 제출 시, 최신 선택 상태가 서버에 전송되는 것을 보장해야 한다
- **FR-005**: 시스템은 터치 이벤트와 마우스 이벤트가 동시에 발생할 때 중복 처리되지 않아야 한다

## Success Criteria _(mandatory)_

### Measurable Outcomes

- **SC-001**: 0.1초 간격으로 10개 날짜를 연속 클릭했을 때 100% 정확한 선택 상태 반영
- **SC-002**: 날짜 선택 직후 0.5초 이내 투표 제출 시, 선택한 모든 날짜가 서버 요청에 포함됨
- **SC-003**: 드래그 선택 시 범위 내 모든 날짜가 누락 없이 선택됨
- **SC-004**: 동일 날짜 더블클릭 시 정확히 선택→해제 토글이 동작함

## Clarifications

### Session 2026-03-23

- Q: 이 버그가 발생하는 주요 시나리오는 무엇인가요? → A: 참여자 투표 날짜 선택과 호스트 모임 생성 날짜 선택 모두에서 발생
- Q: 주로 발생하는 디바이스 환경은 무엇인가요? → A: 모바일(터치)과 데스크톱(마우스) 모두에서 발생

## Assumptions

- 현재 `dragRef`(useRef)와 `dragState`(useState) 이중 상태 관리 패턴에서 비동기 상태 업데이트로 인한 동기화 불일치가 근본 원인으로 추정됨
- React의 useState 배치 업데이트 특성상 빠른 연속 클릭 시 중간 상태가 유실될 수 있음
- `handleDateChange`에서 `setSelectedDates` 호출 후 `formattedDates`(useMemo)가 즉시 갱신되지 않아 제출 시 stale 상태가 전송될 수 있음
